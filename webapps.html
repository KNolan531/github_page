Kool Notes App (React)

This is a webapp created using React that simulates the Google Keeper app. It is basically a note card system in which the user can create a note card and input a title and then the note contents itself. When the user clicks the "+" button then the card is added below and the original note card clears for them to create another new one to add.

The notes line up adjacently similar to a note board concept and can be deleted. This is a basic "front-end" version that clears on page reload and will be updated soon in the future with database/backend capability to save the notes.

This project was created using React and styled using both CSS and material UI react features. The project was created using different components to be rendered: one for the Note card itself, one for the top submit form, and then one for the header/footer.

The usage of Hooks and UseState were implemented to handle the user input in the App component with usage of the Map array function to continuously add more notes. Finally the usage of 2 functions: add and delete in the App component to handle updating the content at run time.

Accounting App (React)

This is a webapp project I completed with a team for SWE App Domain course at Kennesaw State University. This was a semester long project in which we learned concepts relating to a specific domain (Accounting in this case) and then implemented them in different Sprints. Each sprint had it's own requirements that added new features and components to the app.

The Accounting app itself was expected to simulate a real life accounting firm with user types of Admin, Manager and Accountant. Each user type had access to differnet features that consisted of Creating and Modifying financial accounts, creating journal entries and adjusting financial records.

React was chosen as the frontend for this system as it had a User Log-In template/component already easily available and not difficult to work with. Firebase was used to create an online database for the different user accounts using their FireStore system and using API calls to access the data.

Personal Blog App (EJS)

This was a side project I created while taking a course on Embedded Javascript(EJS). It is basically a blog website that lists various blog posts and the user can click on each post to read more and be directed to a new page displaying the whole post. The admin also has access to a Compose Post form to create a new blog post.

This site is implemented using NodeJS, along with template views rendered using Embedded Javascript(EJS) to manipulate the user interface. Different ejs files are created and stored as "views" and directed to when a link is clicked on. A database connection to MongoDB using the Mongoose package is used to save the blog posts so they will always appear on reload.
