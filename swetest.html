Software Testing Course Work

This section will highlight various course assignments and projects from my Software Testing and QA course I took at Kennesaw State. They range from code implementation using JUnit test code to documentation showing different ways to create test cases from Graph to Partition methods.

Each section below has a short description and corresponding button that links to a pdf of the test cases or a package of java junit code.



Graph Coverage

For this assignment we were given Java code "Print Primes" and were asked to use the "Graph Coverage" technique to create test cases. Graph Coverage involves drawing a series of circles and arrows that flow different directions or paths based on the code logic and converting those paths into test cases.

A pdf is included showing my graph for this testing method and the test cases created using various paths. The test cases are then implemented in JUnit and provided below.

Partition Coverage

For this assignment we were given Java code "LimitedList" and used the "Partition Coverage" technique to create test cases. Partition Test Coverage basically involves creating tables for each method and then creating test values for each different applicable variable. These are then converted into test cases.

A pdf is included showing the different tables used and their values and the test cases created using them. The test cases are then implemented in JUnit and provided below.

PDF Java/JUnit Code


Syntax Coverage

For this assignment we were given Java code "PowerTest" and used the "Syntax Coverage" technique to create test cases. This method is based more on the "syntax" of the code as the name implies. In other words testing for mistakes involving a different symbol or invalid value used (mutant code).

A pdf is included showing the different mutants I created for the code with their values and the test cases created using them. The test cases are then implemented in JUnit and provided below.

Final Project

For a final semester project we were asked to create a simple program with different methods(only skeleton code written) to be tested and create test cases for them. We would then give the code to another student who would implement the methods so that the test cases would run successfully in JUnit. The student would then provide feedback on our test cases we created and then use 2 different testing techniques to come up with additional test cases to be added in JUnit.
